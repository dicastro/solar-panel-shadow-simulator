Guía de configuración

Esta guía explica cada sección del fichero config.json utilizado por el Solar Panel Shadow Simulator. Puedes editar la configuración directamente en la aplicación desde Ajustes → Configuración.

1. Ubicación de la instalación

El objeto location define las coordenadas GPS de la instalación. Se utilizan para calcular la posición del sol (altitud y acimut) en cada paso de tiempo simulado.

"location": {
  "latitude": 40.6252,
  "longitude": -4.0141
}
CampoTipoDescripción
latitudenúmeroGrados decimales. Positivo = Norte, negativo = Sur. Rango −90 a 90.
longitudenúmeroGrados decimales. Positivo = Este, negativo = Oeste. Rango −180 a 180.

Define también el campo timezone con el identificador de zona horaria IANA correspondiente a la instalación:

"timezone": "Europe/Madrid"

Puedes encontrar tu identificador de zona horaria en la lista de zonas horarias de la base de datos TZ de Wikipedia. Ejemplos habituales: Europe/Madrid, Europe/London, America/New_York, Asia/Tokyo.

2. Azimut de la instalación

El azimuth de la instalación rota todo el conjunto (muros, barandillas, paneles) en el plano horizontal. Representa la dirección hacia la que apunta la cara frontal de la terraza, medida en grados desde el Sur.

ValorSignificado
0Orientación sur (sin rotación)
+45Rotado 45° hacia el Este (orientación Sur-Este)
-45Rotado 45° hacia el Oeste (orientación Sur-Oeste)
90Orientación Este
-90Orientación Oeste
180 o -180Orientación Norte
Orientación sur — azimuth: 0
Orientación Sur-Este — azimuth: 45

El azimut de los paneles es absoluto, no relativo al azimut de la instalación. Cada array de paneles tiene su propio campo azimuth dentro de arrays[], y utiliza la misma convención (grados desde el Sur). Si tu instalación está orientada al Sur-Este (site.azimuth: 45) y quieres que los paneles también apunten al Sur-Este, define array.azimuth: 45. Si quieres que los paneles apunten al Sur independientemente de la orientación de la instalación, define array.azimuth: 0.

3. Apariencia de la instalación

Estos campos opcionales controlan la apariencia visual de la instalación en la vista 3D. No tienen ningún efecto sobre los cálculos de sombras ni sobre las estimaciones de producción de energía.

CampoTipoPor defectoDescripción
floorColor string (hex CSS) #cccccc Color del suelo de la terraza. Debe ser un color hexadecimal de 6 dígitos. Ejemplos: #b45d16 terracota, #888888 hormigón, #cccccc gris claro. 
"floorColor": "#b45d16"

4. Perímetro de la instalación (puntos de muro)

El array wallPoints define el perímetro de la terraza como una secuencia de coordenadas de esquinas en metros. El sistema de coordenadas es:

  • +X = Este, +Z = Norte (antes de cualquier rotación de la instalación)
  • El origen [0, 0] puede colocarse en cualquier punto — lo que importa son las posiciones relativas
  • Recorre el perímetro en sentido antihorario empezando por la esquina Sur-Oeste

Solo se soportan ángulos de 90° y 180°. Los ángulos no rectos producirán una geometría de muros incorrecta y aparecerá un aviso en la aplicación.

Ejemplo: terraza rectangular (4m × 6m)

Configuración
"wallPoints": [
  [0, 0],
  [4, 0],
  [4, 6],
  [0, 6]
]
Diagrama (vista superior, Norte = arriba)

Las flechas rojas muestran el orden de recorrido antihorario

Ejemplo: terraza en forma de L

Una forma en L requiere 6 puntos. Todos los ángulos interiores deben ser de 90°.

Configuración
"wallPoints": [
  [0, 0],
  [6, 0],
  [6, 4],
  [3, 4],
  [3, 8],
  [0, 8]
]
Diagrama (vista superior, Norte = arriba)

La esquina Sur-Oeste es el mínimo X y el mínimo Z de todos los puntos. Las posiciones de los arrays de paneles (arrays[].position) se miden en metros desde esta esquina.

5. Configuración predeterminada de muros

El objeto wallDefaults define la altura y el grosor aplicados a todos los muros salvo que se sobreescriban individualmente.

"wallDefaults": {
  "height": 0.72,
  "thickness": 0.145
}
CampoUnidadDescripción
heightmetrosAltura del muro sobre el suelo de la terraza.
thicknessmetrosGrosor del muro. Afecta tanto al renderizado como a la geometría de sombras.

6. Sobreescritura de muros individuales

Los muros individuales pueden sobreescribir los valores predeterminados mediante wallsSettings. Cada entrada apunta a un muro por su índice basado en 0 — el índice del punto de inicio de ese segmento en wallPoints. El muro 0 va de wallPoints[0] a wallPoints[1], el muro 1 de wallPoints[1] a wallPoints[2], y así sucesivamente.

"wallsSettings": [
  {
    "wall": 2,
    "override": {
      "height": 1.2
    }
  }
]

Esto eleva el muro 2 (el tercer lado del perímetro) a 1,2 m mientras todos los demás muros mantienen la altura predeterminada. Puedes sobreescribir height y/o la configuración de barandilla de cualquier muro individual.

7. Barandillas

Las barandillas se colocan sobre los muros. El objeto railingDefaults define el estilo global de barandilla; los muros individuales pueden sobreescribirlo mediante wallsSettings[].override.railing.

"railingDefaults": {
  "active": true,
  "heightOffset": 0.185,
  "shape": { "kind": "cylinder", "radius": 0.025 },
  "support": {
    "shape": { "kind": "cylinder", "radius": 0.008 },
    "count": 3,
    "edgeDistance": 0.22
  },
  "extendAtStart": true,
  "extendAtEnd": true,
  "extensionGap": 0.04
}
CampoTipo / valoresDescripción
activebooleanoSi se renderiza barandilla en este muro.
heightOffsetmetrosDistancia desde la parte superior del muro hasta el centro del pasamanos.
shapever más abajoSección transversal del pasamanos.
supportver más abajoBalaustres que sostienen el pasamanos.
extendAtStart / extendAtEndbooleanoSi el pasamanos se extiende sobre el poste de esquina en cada extremo, para un encuentro de esquina a ras.
extensionGapmetrosSeparación entre dos extensiones de pasamanos que se encuentran en un poste de esquina.

Formas del pasamanos

Cilindro (barra redonda)
"shape": {
  "kind": "cylinder",
  "radius": 0.025
}
Cuadrado (barra plana)
"shape": {
  "kind": "square",
  "width": 0.04,
  "height": 0.008
}
Sin barandilla en un muro concreto
"wallsSettings": [{
  "wall": 0,
  "override": {
    "railing": { "active": false }
  }
}]

Configuración de balaustres

CampoDescripción
shapeSección transversal: cylinder (con radius) o square (con width y depth).
countNúmero de balaustres por muro. Mínimo 2.
edgeDistanceDistancia desde cada extremo del muro hasta el balaustre más cercano. Si se omite, los balaustres se distribuyen uniformemente.

8. Pérdidas del sistema

Estos tres campos opcionales en site modelan las pérdidas de energía reales entre los paneles y el punto de conexión a la red.

"groundAlbedo": 0.20,
"inverterEfficiency": 0.97,
"wiringLoss": 0.02
CampoValor por defectoDescripción
groundAlbedo0.20Fracción de la irradiancia global horizontal reflejada por el suelo hacia la cara inferior del panel. 0.20 = hormigón/asfalto. 0.25 = grava clara. 0.10 = membrana de cubierta oscura.
inverterEfficiency0.97Eficiencia de conversión CC/CA del inversor. Inversores de cadena modernos típicos: 0.96–0.98.
wiringLoss0.02Fracción de potencia CC perdida en el cableado entre paneles e inversor. Instalaciones residenciales típicas: 0.01–0.03.

9. Configuraciones (setups)

Una configuración (setup) es un conjunto completo de paneles a simular. Puedes definir varias configuraciones en el array setups para comparar distintos diseños, orientaciones o equipos.

"setups": [
  { "label": "Instalación actual", ... },
  { "label": "Con optimizadores en fila 1", ... }
]

Cada configuración tiene un label (mostrado en la interfaz y en los informes), un bloque panelDefaults y uno o más arrays.

10. Propiedades predeterminadas de paneles

El objeto panelDefaults define las características físicas y eléctricas de los paneles utilizados en todos los arrays de esta configuración, salvo sobreescritura a nivel de array.

"panelDefaults": {
  "width": 1.134,
  "height": 2.094,
  "peakPower": 415,
  "zones": 3,
  "zonesDisposition": "horizontal",
  "hasOptimizer": false,
  "string": "S1",
  "temperatureCoefficient": -0.0036,
  "noct": 44
}
CampoUnidad / valoresDescripción
widthmetrosAnchura del panel según la ficha técnica.
heightmetrosAltura del panel según la ficha técnica.
peakPowerWpPotencia pico en Condiciones de Test Estándar (1000 W/m², 25°C).
zonesentero ≥ 1Número de zonas con diodo de bypass. La mayoría de paneles residenciales tienen 2 o 3 zonas.
zonesDispositionhorizontal / verticalCómo se dividen las zonas en la cara del panel. horizontal = apiladas de arriba a abajo (lo más habitual). vertical = una al lado de la otra.
hasOptimizerbooleanoSi el panel lleva un optimizador de potencia CC (p.ej. SolarEdge). Los optimizadores eliminan la penalización por desajuste de string.
stringcadenaIdentificador del string al que pertenece este panel. Todos los paneles del mismo string comparten la misma limitación de corriente en serie. Ejemplo: "S1", "S2".
temperatureCoefficient/°CVariación relativa de la potencia pico por grado por encima de 25°C. Se encuentra en la ficha técnica como coeficiente de temperatura de Pmax. Valor típico: −0.004 /°C. Valor por defecto si se omite: −0.004.
noct°CTemperatura Nominal de Operación de la Célula. Se encuentra en la ficha técnica. Se usa para estimar la temperatura real de la célula a partir de la temperatura ambiente y la irradiancia. Valor por defecto si se omite: 45.

Zonas y diodos de bypass

Un diodo de bypass protege una zona. Cuando una zona queda completamente sombreada, su diodo se activa y esa zona produce cero potencia, pero el resto del panel continúa funcionando. La simulación cuenta cuántos puntos de muestreo de cada zona están sombreados y aplica un umbral configurable para decidir si la zona se considera bloqueada.

zonesDisposition: "horizontal" es la disposición más habitual — el panel se divide en bandas horizontales que recorren su anchura. Esto coincide con cómo están físicamente cableados con diodos de bypass la mayoría de los paneles.

11. Arrays de paneles

Cada configuración contiene uno o más arrays. Un array es una cuadrícula rectangular de paneles idénticos en una posición, inclinación y acimut fijos.

"arrays": [
  {
    "position": [1.5, 1.0],
    "azimuth": 0,
    "elevation": 0.10,
    "inclination": 20,
    "rows": 3,
    "columns": 2,
    "spacing": [0.02, 0.04],
    "orientation": "portrait"
  }
]
CampoUnidad / valoresDescripción
position[metros Este, metros Norte]Distancia desde la esquina SO de la instalación hasta la esquina SO de este array.
azimuthgradosAcimut absoluto de los paneles (0 = Sur, +Este, −Oeste). Independiente del acimut de la instalación.
elevationmetrosAltura del borde inferior del array sobre el suelo de la terraza.
inclinationgradosÁngulo de inclinación respecto a la horizontal. 0° = plano. 90° = vertical. Residencial típico: 10–35°.
rowsenteroNúmero de filas de paneles. La fila 0 es la más al Sur; las filas aumentan hacia el Norte.
columnsenteroNúmero de columnas. La columna 0 es la más al Oeste; las columnas aumentan hacia el Este.
spacing[horizontal m, vertical m]Separación entre paneles adyacentes. Opcional; por defecto [0.02, 0.02].
orientationportrait / landscapeSi los paneles se montan más altos que anchos o más anchos que altos. Opcional; por defecto portrait.

Las sobreescrituras a nivel de array — width, height, peakPower, zones, zonesDisposition, hasOptimizer, string, temperatureCoefficient, noct — pueden definirse directamente en el objeto del array para sobreescribir los panelDefaults de la configuración.

Ejemplo: dos arrays en diferentes strings

"arrays": [
  {
    "position": [0.5, 1.0],
    "azimuth": 0, "elevation": 0.10, "inclination": 18,
    "rows": 2, "columns": 3,
    "string": "S1"
  },
  {
    "position": [0.5, 4.5],
    "azimuth": 0, "elevation": 0.10, "inclination": 18,
    "rows": 1, "columns": 3,
    "string": "S2"
  }
]

12. Sobreescritura de paneles individuales

Los paneles individuales dentro de un array pueden sobreescribirse mediante arraysSettings. Cada entrada apunta a un panel concreto por su índice de array, fila y columna.

"arraysSettings": [
  {
    "array": 0,
    "row": 0,
    "col": 2,
    "hasOptimizer": true,
    "string": "S2"
  }
]

Esto mueve el panel del array 0, fila 0, columna 2 al string S2 y le añade un optimizador. El resto de propiedades se heredan de los valores predeterminados del array. Se pueden listar varias entradas para sobreescribir varios paneles.

Los paneles con optimizador se muestran en verde en la vista 3D; los paneles sin optimizador se muestran en azul oscuro.

    }

    {
      "label": "Sur 20grados 2x4",
